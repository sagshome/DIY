Data Views

While the default view for looking at account or portfolio data is the chart view, this the Data View is where most of the work happens. The general idea behind the Data View is to provide a month-over-month summary and provide a mechanism to correct data. Data usually only needs to be correct if transactions were missing or incorrect.

Account Types

The data view is tuned based on account types.

  • Bank Accounts - You simply edit/record the balance each month
  • Value A little more complicated than a bank account. You still do the monthly value updates, but you can add any funds deposited or withdrawn in that time period. This allows us to track return on investment
  • Investment Accounts hold multiple equities, so we provide a button to update indivdual account entities. Most Equities/ETFs we can look up via our APIs so we can calculate value (price * shares) and even find out if dividends were paid.

    • Portfolios vs Accounts

    The data will look the same, but you can not reconcile at the portfolio level. You must tunnel down to an account
